# 《设计概论》7次课PPT脚本总索引

## 目标

依据《设计概论》（叶芳、吕杰锋）扫描版教材，将全书规划为7次课，每次160分钟，并明确每一页PPT的课程内容。

## 两条内容主线

1. 教材主线：客观讲授教材中的概念、分类、历史、方法、职业与管理知识。
2. 当下与未来主线：用当前标准、数字技术、人工智能、可持续发展和行业变化解释教材，并提示未来学习方向。

## 文件清单

1. `lesson01-绪论与设计的历程.md`：绪论、第一章，含1页课程总封面。
2. `lesson02-产品设计.md`：第二章。
3. `lesson03-环境设计.md`：第三章。
4. `lesson04-传达设计.md`：第四章。
5. `lesson05-设计程序与方法.md`：第五章。
6. `lesson06-设计师与职业发展.md`：第六章、第七章、第八章。
7. `lesson07-设计管理与创新创业.md`：第九章、第十章。
8. `current-sources.md`：当下知识与未来趋势的公开来源。

## 统一使用方法

- 每次课分成4个40分钟单元，总计160分钟。
- 每页只承担一个核心结论，避免把讲义整段复制到PPT。
- “教材讲授”是课堂事实基础；“当下与未来”是扩展，不替代教材。
- 建议教师根据学生专业方向替换案例图片，但不要改变章节知识结构。
- 标注“互动”“练习”“讨论”的页面应真实预留课堂时间，不作为快速翻页页。

## 教材可视化要求

- 所有页面优先把教材文字转为图示、图例、图表或结构化对照，不直接堆段落。
- “课堂动作与视觉建议”一列不是装饰建议，而是该页知识的推荐表达形式。
- 图解必须准确对应教材原有逻辑，不能制作只有版式效果、没有知识含义的伪信息图。
- 当教材逻辑本身是并列、递进、因果、演化、空间层级或系统关系时，页面表达也必须保持同一种逻辑。

推荐匹配规则：

- 定义、构成、分类：概念结构图、树状图、谱系图。
- 因果关系：因果链、关系网络、反馈回路图。
- 发展历程：时间轴、阶段演化图。
- 方法步骤：流程图、决策路径图。
- 空间层级：尺度嵌套图、地图、剖面图、空间模型。
- 对比内容：比较表、矩阵图、坐标图。
- 系统关系：系统模型、输入—过程—输出图。
- 数据与结论：柱状图、折线图、比例图、重点数字。
- 综合理论：知识地图、框架图。

## 建议页面结构

- 标题：一句结论或一个明确问题。
- 主体：3—5个要点，优先图示、时间轴、矩阵、流程图和案例对照。
- 页脚：教材章节与页码；涉及当下资料时标注 `[当下1]` 等来源编号。
- 讲述：先讲教材定义，再讲现实案例，最后提出未来问题。

## 核对结果

- 第1次课：33页，160分钟。
- 第2—7次课：每次32页，160分钟。
- 全课程：225页，1120分钟。
